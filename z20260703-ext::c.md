# Bridging C to C3, the libc C headers

Among other low level languages, C3 language differentiates itself by minimal syntactic changes from C. My first thought was "Great! So interoperability with C should also be easier, at least, as easy as C++." However, as I continue to code in C3, I felt two non-trivial troubles. (1) C3's strict naming rule drags me back, whenever I try to translate C codes. (2) And my C-baked brain constantly wants to use common libc functions.

C3's naming rule itself is quite clean -- CONSTANT or FAULT in uppercase, variables and function/macros in snake_case or camelCase, Types or Structs in PascalCase. Beautiful, but I constantly had to deal with the random naming issues, when I wanted to call existing C codes which are not conforming to the naming rule. This kind of breaking change has its well intended reason, and so just accept it.

For libc functions, current C3 std lib provides `libc` module, but I immediately found that C3's current std `libc` covers only a fraction of C libc, and I had to define `extern` functions quite often here and there. I constantly wanted to use common C header files like `#include <string.h>` or `#include <sys/socket.h>`... as natural as C. So I thought, "what if I convert all common 'C-libc' headers for C3?" 

As soon as I started off, I realized that it's a daunting task, because (1) there are multiple libc's -- glibc, musl libc, uClibc, Bionic, Newlib, msvcrt, (2) there are thousands of functions to be mapped, (3) different OS'es and machine architectures provide their own version of C headers. ISO C standard requires 200 to 300 functions like `printf, malloc, strcpy`. POSIX Standard requires more than 1100 functions like `fork, open, pthread_create`. Different implimentation provides different functions for their different goals. `glibc` exports 2500+ functions. `musl libc`/`bionic libc` are known to provide 1200+ functions. `Newlib` covers roughly 300 core functions only.

So I decided to convert all headers that I actually bump into, along the way of building an extended library of C3. Here is the list of converted header files.

All of them are stuffed into a single flat directory `ext/c` with the extention of `*.h.c3`. So you can easily copy and use them by including `ext/c/*.h.c3` in your compilation.

Import the modules you need like this:

```c3
import c::stdio;
import c::unistd;
import c::sys::socket;
import c::errno;
```

Call C functions through their C3 module namespace:

```c3
import c::stdio;
import c::string;
import c::stdlib;

fn void main(String[] args)
{
    if (args.len != 2) {
        stdio::printf("Usage: %s 'arg1'\n", args[0].zstr_tcopy());
        stdlib::exit(0);
    }
    
    if (string::strcmp(args[1].zstr_tcopy(), "OK") == 0) {
        stdio::printf("Hello from ext::c! OK\n");
    } else {
        stdio::printf("Hello from ext::c! %d\n", string::strlen(argv[1].zstr_tcopy()));
    }
}
```

* Note that `char*` of C is normally a zero-terminated string, which is mapped to `ZString` in C3. Default C3 `String` doesn't guarantee zero-termination.

### POSIX Example

```c3
import c::sys::socket;
import c::netinet::in;
import c::unistd;
import c::errno;

fn int? create_server_socket(ushort port)
{
    int sockfd = socket::socket(socket::AF_INET, socket::SOCK_STREAM, 0);
    if (sockfd < 0) return errno::get_fault()~; // POSIX/Windows `errno` to C3 `fault`

    /*
     * bind(), listen(), and other socket setup code would follow here.
     */

    return sockfd;
}
```

Here you see `socket::` prefix in every function and CONSTANT.

* Note that `c::errno` module provides `errno::get_fault()` which returns a proper C3 fault corresponding to the `errno` value of previous system call.

### Windows Example

```c3
import c::winsock2;
import c::ws2tcpip;

fn void? init_networking()
{
    WSAData wsa_data;

    int rc = winsock2::wsaStartup(0x0202, &wsa_data);
    if (rc != 0) return winsock2::wsa_fault()~; // wsaGetLaseError() to C3 `fault`
}
```

* Note that we have to call `wsaStartup()` before any Winsock functions.

* Note that on Windows, there are two ways to get `errno` (`getLastError(), wsaGetLaseError()`, and so there is anothe function `errno::wsa_fault()` to get a C3 fault after Winsock related system calls.

### Naming Differences from C

C3 has stricter naming rules than C, so these bindings cannot always preserve C names exactly as C3 identifiers. The original C symbol name is preserved with `@cname()` when needed.

General rules used by this package:

- Types and structs use **PascalCase**
- Variables and functions/macros use **snake_case or camelCase**
- Constants and faults use **UPPERCASE**
- C function names that do not follow C3 naming rules are mapped with `@cname()`

Example:

```c
/* C: winsock2.h */

struct WSADATA
{
};

int WSAStartup(unsigned short version_required, struct WSADATA* wsa_data);
```

```c3
// C3: winsock2.h.c3

struct WSAData @cname("WSADATA")
{
}

extern fn int wsaStartup(ushort version_required, WSAData* wsa_data)
    @cname("WSAStartup");
```

### Compatibility

This collection of C headers are mean to be used as like in C. Depending on the coverage, the modules are attributed like:
* `@if(env::POSIX)` for POSIX systems
* `@if(env::WIN32)` for Windows 
* `@if(env::LINUX)` for Linux 
* `@if(env::DARWIN)` for macOS 
* `@if(env::FREEBSD), @if(env::NETBSD), @if(env::NETBSD), @if(env::OPENBSD)` for BSD family
* `@if(env::X86_64)` for Intel/AMD 64bit systems,
* `@if(env::AARCH64` for ARM 64bit systems
